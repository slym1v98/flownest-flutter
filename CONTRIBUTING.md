# Contributing to Kappa

Thank you for your interest in contributing to Kappa! We welcome contributions in many forms, including bug reports, feature requests, documentation improvements, and code contributions.

## How to Contribute

### 1. Reporting Bugs
*   Ensure the bug was not already reported by searching on GitHub Issues.
*   If you're unable to find an open issue addressing the problem, open a new one.
*   Be as detailed as possible, including steps to reproduce the issue, expected vs. actual behavior, and relevant logs or screenshots.

### 2. Feature Requests
*   Search for existing feature requests.
*   Open a new issue describing the feature, the use case, and any potential implementation details.

### 3. Documentation Improvements
*   Documentation is key to any project's success. If you find typos, missing information, or unclear explanations, please open a PR.

### 4. Code Contributions
*   Fork the repository.
*   Create a new branch for your feature or bug fix.
*   Write clear, concise code that follows the project's existing style.
*   Include unit tests for any new functionality or bug fixes.
*   Submit a Pull Request (PR) with a detailed description of your changes.

## Development Setup

1.  Clone the repository: `git clone https://github.com/your-repo/kappa.git`
2.  Install dependencies: `flutter pub get`
3.  Run tests: `flutter test`

## Code Style

We follow the official [Dart Style Guide](https://dart.dev/guides/language/analysis-options). Ensure your code passes `flutter analyze` before submitting a PR.
